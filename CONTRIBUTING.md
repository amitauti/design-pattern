# Contributing

Thanks for contributing! Please follow these guidelines:

- Create or update a pattern under `patterns/high-level` or `patterns/low-level`.
- Use `templates/pattern-template.md` for new pattern pages, ensuring all sections are completed: Summary, Problem, Context, Solution, Diagram(s), Examples, References, Guardrails, Monitoring, Governance, Security, Testing, Reproducibility, Lifecycle, and Delivery Checklist.
- Include at least one diagram in Mermaid or PlantUML format.
- Keep content concise and include references.
- Open a PR with a descriptive title and link to related issues.
